# **Step-by-Step Tutorial: Importing JSON into MongoDB with Authentication and Ensuring Data Persistence**

This tutorial provides step-by-step instructions to import a JSON file (`lightswitch.json`) into a MongoDB database (`lightswitch`) using the terminal. It also covers authentication issues, data persistence, and how to resolve them.

---
## **1. Verify MongoDB Authentication**

Check if MongoDB requires authentication:
```bash
docker exec -it docker-mongodb-1 mongosh --eval 'db.runCommand({ connectionStatus: 1 })'
# docker exec -it mongodb mongosh --eval 'db.runCommand({ connectionStatus: 1 })'
```
- If `authInfo` contains credentials, authentication is enabled.
- If `authInfo` is empty, authentication is **not enabled**.

---
## **2. Import JSON Data into MongoDB**

Run the following command to import `lightswitch.json` into the `lightswitch` database:
```bash
docker exec -i docker-mongodb-1 mongoimport --db lightswitch --collection lightswitch \
  --file /lightswitch.json --username root --password password --authenticationDatabase admin
```

To verify the import:
```bash
docker exec -it mongodb mongosh --username root --password password --authenticationDatabase admin --eval 'db.lightswitch.find().pretty()'
```

If authentication fails, continue with the next steps to fix permissions.

---
## **3. Fix "Unauthorized: Command listCollections requires authentication" Error**

If running `show collections` results in an authentication error, follow these steps.

### **3.1 Log in as Root**
```bash
docker exec -it mongodb mongosh --username root --password password --authenticationDatabase admin
```

### **3.2 Check User Roles**
```javascript
db.getUser("root")
```
If the output does not include **readWrite** permissions for `lightswitch`, grant them.

### **3.3 Grant Permissions to Root User**
Switch to the `admin` database:
```javascript
use admin
```
Grant **readWrite** permissions for `lightswitch`:
```javascript
db.grantRolesToUser("root", [{ role: "readWrite", db: "lightswitch" }])
```

### **3.4 Verify Permissions**
Switch back to `lightswitch`:
```javascript
use lightswitch
show collections
db.lightswitch.find().pretty()
```

If successful, your import and authentication settings are correct.

---
## **4. Alternative: Create a New User for `lightswitch`**

Instead of modifying `root`, you can create a new user with read/write access.

### **4.1 Create a New User**
```javascript
use lightswitch
db.createUser({
    user: "lightswitch_user",
    pwd: "securepassword",
    roles: [{ role: "readWrite", db: "lightswitch" }]
})
```

### **4.2 Log in as `lightswitch_user`**
```bash
docker exec -it mongodb mongosh --username lightswitch_user --password securepassword --authenticationDatabase lightswitch
```
### **4.3 Verify Collections and Data**
```javascript
show collections
db.lightswitch.find().pretty()
```

---
## **5. Ensure MongoDB Data Persistence**

### **5.1 Verify If MongoDB Data is Persisting**
Run:
```bash
docker volume ls
```
Ensure the following volumes exist:
```
local     docker_benchmark_mongo-config
local     docker_benchmark_mongo-data
```

### **5.2 Insert and Verify Data Persistence**
1. **Insert Test Data**
   ```bash
   docker exec -it mongodb mongosh --username root --password password --authenticationDatabase admin
   ```
   Inside MongoDB:
   ```javascript
   use lightswitch
   db.lightswitch.insertOne({ "test": "data" })
   db.lightswitch.find().pretty()
   ```

2. **Restart MongoDB**
   ```bash
   docker restart docker-mongodb-1
   ```

3. **Check If Data Still Exists**
   ```bash
   docker exec -it mongodb mongosh --username root --password password --authenticationDatabase admin
   ```
   ```javascript
   use lightswitch
   db.lightswitch.find().pretty()
   ```
   - ✅ If `{ "test": "data" }` appears, persistence is working.
   - ❌ If data is missing, check if the correct volumes are mounted.

### **5.3 Check Mounted Volumes Inside the Container**
```bash
docker inspect mongodb | grep "Mounts"
```
Expected output:
```json
"Source": "/var/lib/docker/volumes/docker_benchmark_mongo-data/_data",
"Destination": "/data/db",
"Source": "/var/lib/docker/volumes/docker_benchmark_mongo-config/_data",
"Destination": "/data/configdb",
```
If missing, MongoDB is not using the correct volumes.


If `--jsonArray` is needed, your JSON file likely contains an array of objects.

---
## **Conclusion**
- If you face authentication errors, grant proper permissions or create a new user.
- Always verify imports using `find().pretty()`.
- Ensure MongoDB data persists across restarts using volume checks.
- Use `show collections` and `countDocuments()` to check data presence.

This tutorial ensures you can reliably import `lightswitch.json` into MongoDB and maintain data persistence in future setups. 🚀
